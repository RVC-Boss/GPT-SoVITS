# 贡献指南

感谢你对 **RVC-Boss/GPT-SoVITS** 项目的兴趣！我们欢迎所有形式的贡献，包括改进文档、修复 bug、新功能开发等。以下是贡献的基本流程：

## 部署 GPT-SoVITS

按照说明部署 GPT-SoVITS。如果您在部署 GPT-SoVITS 时遇到困难，请查看 [FAQ](./FAQ.md) 以获取常见解决方案。


## 如何贡献

### 1. Fork 仓库

点击页面右上角的 **Fork** 按钮，将该仓库 Fork 到你的 GitHub 账户。

### 2. 克隆你的 Fork

在本地克隆你 Fork 的仓库：

```bash
git clone https://github.com/你的用户名/GPT-SoVITS.git
cd GPT-SoVITS
```

### 3. 创建分支

为你的改动创建一个新的分支：

```bash
git checkout -b feature/你的分支名
```

### 4. 进行改动

在你的分支上进行代码修改或其他改动。请确保遵循项目的代码风格和规范。

### 5. 提交改动

将你的改动提交到本地仓库：

```bash
git add .
git commit -m "描述你的改动"
```

### 6. 推送到 GitHub

将你的改动推送到你的 Fork 仓库：

```bash
git push origin feature/你的分支名
```

### 7. 创建 Pull Request

在 GitHub 上访问你的 Fork 仓库，点击 **New Pull Request** 按钮提交你的改动。

请在 Pull Request 中详细描述你的改动内容和目的，可以提供少量测试用例以便复现。如果可能，请为在 Commit_Message 中添加一些说明，方便项目的维护。

## 代码规范

- 请保持代码简洁和注释清晰。
- 请遵循 [PEP 8](https://www.python.org/dev/peps/pep-0008/) 的 Python 代码风格指南。
- 请确保你的改动通过所有现有的测试，并添加适当的新测试。
